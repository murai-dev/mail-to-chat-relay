## Gmail設定

### ステップ1: IMAPを有効にする

1. ブラウザでGmailにアクセス: https://mail.google.com
2. 右上の歯車アイコン → **「すべての設定を表示」** をクリック
3. **「転送とPOP/IMAP」** タブをクリック
4. **「IMAP」** セクションで「**IMAPを有効にする**」を選択

   **スクリーンショット**: 設定画面のIMAP有効化ラジオボタン

5. ページ下部の **「変更を保存」** をクリック

### ステップ2: アプリパスワードを生成

1. Googleアカウント管理ページにアクセス: https://myaccount.google.com
2. 左メニューの **「セキュリティ」** をクリック
3. ページを下にスクロールして **「アプリ パスワード」** を見つける

   **注意**: 「アプリ パスワード」がない場合は、2段階認証が有効になっていません。以下の「2段階認証の設定」を先に実施してください。

4. **「アプリ パスワード」** をクリック
5. 「デバイスを選択」で **「その他（カスタム名）」** を選択
6. 「**Mail Relay**」などの名前を入力
7. **「生成」** ボタンをクリック

   **スクリーンショット**: アプリパスワード生成画面（16文字のパスワードが表示される）

8. **黄色いボックスに表示された16文字のパスワードをコピー** （スペースは無視）

   **警告**: このパスワードは後ほど ``.env`` ファイルに貼り付けます。安全に保管してください。

### 2段階認証の設定（アプリパスワード設定時に必要な場合）

1. https://myaccount.google.com にアクセス
2. 左メニューの **「セキュリティ」** をクリック
3. **「2段階認証プロセス」** をクリック
4. 画面に従って設定完了

## Discordウェブフック設定

### ステップ1: 通知先チャンネルを確認

1. **Discord** アプリ/ウェブを開く
2. 通知を受け取りたい **サーバーとチャンネル** を確認

   **スクリーンショット**: Discordのチャンネルリスト（#notifications など）

### ステップ2: ウェブフックを作成

1. 通知先チャンネルを右クリック → **「チャンネル設定」** をクリック

   **スクリーンショット**: チャンネル右クリックメニュー

2. 左メニューから **「連携サービス」** をクリック
3. **「ウェブフック」** をクリック
4. **「新しいウェブフック」** ボタンをクリック

   **スクリーンショット**: ウェブフック作成画面

5. ウェブフック名を入力（例:「Mail Relay」）
6. **「ウェブフックを作成」** をクリック

### ステップ3: ウェブフック URLをコピー

1. 作成されたウェブフックの **「URLをコピー」** ボタンをクリック

   **スクリーンショット**: ウェブフックURL表示画面

2. コピーした URLを安全に保管

   **警告**: このURLは後ほど ``.env`` ファイルに貼り付けます。他人に共有しないでください。

## 設定ファイルの準備

### ステップ1: ``.env`` ファイルを作成

1. プロジェクトフォルダをファイラーで開く
2. 新しいテキストファイルを作成
3. ファイル名を `.env` に変更

   **注意**: `.env` は拡張子なし、ドット付きのファイル名です

4. 以下の内容を記入:

```
MAIL_USER=あなたのGmailアドレス@gmail.com
MAIL_PASSWORD=Gmailで生成した16文字のアプリパスワード
DISCORD_WEBHOOK_URL=Discordウェブフックから取得したURL
```

**例**:
```
MAIL_USER=yamada@gmail.com
MAIL_PASSWORD=abcd efgh ijkl mnop
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/123456789/abcdefghijk
```

5. ファイルを保存

### ステップ2: `settings.yaml`` を確認（必要に応じて編集）

1. `config/settings.yaml` をテキストエディタで開く
2. メールフィルター条件を確認

   ```yaml
   filters:
     - type: "subject"
       condition: "prefix"
       value: "【重要】"
   ```

   このセクションで、以下を指定します：
   - `value`: 件名の何を検出するか（例：「【重要】」で始まるメール）

3. 必要に応じて値を変更して保存

### ステップ3: 実行間隔を確認（オプション）

`settings.yaml` の以下の部分で実行間隔を確認：

```yaml
mail:
  check_interval_minutes: 30  # 30分ごとにチェック
```

通常は **30分** で問題ありません。変更したい場合は以下の値を選択してください：
- `15`: 15分ごと
- `30`: 30分ごと（推奨）
- `60`: 60分ごと（1時間ごと）

## システム起動

### ステップ1: コマンドラインを開く

1. **Macの場合**: Spotlight検索で「ターミナル」を開く
2. **Windowsの場合**: PowerShellまたはコマンドプロンプトを開く

### ステップ2: プロジェクトフォルダに移動

```bash
cd /path/to/mail-to-chat-relay
```

**例**:
```bash
cd ~/develop/mail-to-chat-relay
```

### ステップ3: システムを起動

```bash
docker-compose up -d
```

このコマンドでシステムがバックグラウンドで起動します。

**スクリーンショット**: コマンドラインの実行例

## 動作確認

### ステップ1: ログを確認

```bash
docker-compose logs -f
```

以下のようなログが表示されれば成功です：

```
[*] crontab を生成中...
[*] 生成された crontab:
    */30 * * * * python3 /app/src/main.py >> /var/log/mail-relay.log 2>&1
[✓] crontab を生成しました
[*] cron デーモンを起動中...
```

**スクリーンショット**: 起動成功時のログ画面

### ステップ2: テストメールを送信

1. Gmailで、設定した条件を満たすメールを自分宛に送信
   - 例：件名が「【重要】テスト」のメール

   **スクリーンショット**: Gmail送信画面

2. 最大 **2〜3分** 待つ

3. Discordの対象チャンネルで通知が届いているか確認

   **スクリーンショット**: Discord通知受信画面

### ステップ3: 動作しない場合

- **Discordに通知が来ない場合**:
  1. Webhookチャンネルの設定を確認
  2. `DISCORD_WEBHOOK_URL` が正しくコピーできているか確認（URLの最後の文字まで）
  3. `docker-compose logs` でエラーメッセージを確認

- **メールが検出されない場合**:
  1. 件名が `settings.yaml` の設定に合致しているか確認
  2. メールの受信時刻を確認（テスト日時より古いメールは対象外）
  3. `MAIL_USER` と `MAIL_PASSWORD` が正しいか確認

### よくある質問

**Q1: 設定を変更したい場合は？**  
A: `settings.yaml` または `.env` を編集後、以下を実行してください：
```bash
docker-compose restart
```

**Q2: システムを停止したい場合は？**  
A: 以下を実行してください：
```bash
docker-compose down
```

**Q3: ログをすべて確認したい場合は？**  
A: 以下を実行してください：
```bash
docker-compose logs
```
